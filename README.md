# 🌍 IUCN Species Dashboard

**What if every one of the 172,620 species on the IUCN Red List had a voice?**
**The Awareness Engine is the infrastructure to make that possible.**

The IUCN Red List documents 172,620 species. Most of them will go extinct unnoticed — not because nobody cares, but because nobody told their story.
The **IUCN Species Dashboard** is the public verification layer of the Awareness Engine ecosystem — demonstrating that the data foundation is real, current, and scientifically sound.

## 🎯 What This Dashboard Does — and Why It Exists

This dashboard serves two explicit purposes:

**1. Verification Layer**
The IUCN Red List API v4 is the **Single Source of Truth (SSoT)**. The dashboard makes this visible — providing a public, auditable proof that the Awareness Engine's data foundation is accurate, complete, and validated against official IUCN statistics.

```
IUCN Red List API v4 (SSoT)
        │
        ▼
Awareness Engine Backend
        │
        ├── JSON Snapshot ──► Dashboard (Verification Layer)
        │                     "The data is real. Here is the proof."
        │
        └── Live API Query ──► Narrative Generation
                               "The story is current. Every time."
```

**2. Proof of Concept for Thematic JSON Distribution**
The JSON snapshot that powers this dashboard was generated by the Awareness Engine backend. This demonstrates a key architectural capability: **any thematic slice of the IUCN Red List can be exported as a structured, validated JSON file** — ready for use in any frontend, campaign tool, or open data initiative.

```
Awareness Engine Backend
        │
        ├── [Snapshot: All Categories]        ──► This Dashboard
        ├── [Snapshot: Critically Endangered] ──► NGO Campaign Tool
        ├── [Snapshot: Marine Species]        ──► #NatureForAll Frontend
        ├── [Snapshot: African Fauna]         ──► Regional Education App
        └── [Snapshot: Any Theme]             ──► Hackathons / Developer Contests
                                                  "172,620 species. Real data.
                                                   Build something that matters."
```

## 🏗 Architecture of Resilience

This project is built with a "Conservation-First" mindset, ensuring that data is always available, even when external systems are not.

1. **Official API Integration:** Built as a tribute to the scientific rigour of the IUCN. The project uses an official API key to ensure authorized and respectful data retrieval.

2. **JSON Snapshot Engine:**
   - **Generated by the Awareness Engine backend** — the same system that powers narrative generation also produces structured JSON datasets for verification and distribution.
   - **Fail-safe Design:** The dashboard operates on localized JSON snapshots. If the live API undergoes structural changes or is temporarily unreachable, the system remains fully operational using the last "Golden Snapshot."
   - **Resource Efficiency:** This architecture minimizes redundant requests to IUCN servers, providing lightning-fast load times while being a responsible consumer of their infrastructure.

3. **Data Auditing:** The snapshot layer provides a transparent "sanity check" — category counts and species totals are validated against official IUCN statistics before any data enters the narrative pipeline. A ✓ Match confirmation is displayed publicly.

> **Key distinction:** The dashboard reads from a validated JSON snapshot generated by the Awareness Engine backend. The Awareness Engine itself makes **live requests** directly to the IUCN Red List API v4 for narrative generation — ensuring every story is based on the most current species data available.

## 📊 Strategic Features

The dashboard provides more than just individual species facts; it offers a macro-view of global biodiversity:

- **Global Scale Visualization:** Provides an immediate overview of the **172,620+ assessed species**
- **Category Mapping:** Visualizes the distribution across all IUCN categories (from _Extinct_ to _Least Concern_)
- **Quantified Insight:** Users can see exactly how many species are assigned to each threat level, highlighting where communication and conservation efforts are most urgently needed
- **Validation Confirmation:** A public ✓ Match confirms that snapshot data aligns with official IUCN statistics — scientific integrity made visible

## 💡 Dashboard & Engine: Two Roles, One Mission

```
┌──────────────────────────────────────────────────────────────┐
│                   IUCN Red List API v4                       │
│              Single Source of Truth (SSoT)                   │
└──────────────┬───────────────────────────┬───────────────────┘
               │                           │
               ▼                           ▼
  ┌────────────────────┐       ┌────────────────────────┐
  │  JSON Snapshot     │       │  Live API Query        │
  │  (periodic)        │       │  (per species)         │
  └────────┬───────────┘       └────────────┬───────────┘
           │                                │
           ▼                                ▼
  ┌────────────────────┐       ┌────────────────────────┐
  │  Dashboard         │       │  Awareness Engine      │
  │  Verification      │       │  (local development)   │
  │  Layer             │       │                        │

  │   is real."        │       │   is true."            │
  └────────────────────┘       └────────────────────────┘
```

The dashboard manages the **"What"** (verified facts at scale).
The Awareness Engine generates the **"Why"** (the emotional narrative per species).

By using clean, resilient JSON data, the Engine creates tailored content for:

- **Nature-based Education** (Schools & Youth)
- **SBCC** (Social & Behavioural Change Communication)
- **#NatureForAll Campaigns**
- **Reimagining Conservation Communications**

The Awareness Engine is intended to be developed **for and with IUCN** — not as a product, but as a partnership. The data is theirs. The mission is shared. The tool belongs in their hands first.

## 🛠 Tech Stack

- **Frontend:** Next.js / React (deployed on Vercel)
- **Backend:** Node.js / Express (local development)
- **Data Source:** Official IUCN Red List API v4 — Single Source of Truth
- **JSON Distribution:** Awareness Engine backend generates validated thematic snapshots
- **LLMs:** DeepSeek, Google Gemini (multi-LLM architecture for resilience and vendor independence)
- **Resilience:** Decoupled JSON snapshot architecture with public validation layer
